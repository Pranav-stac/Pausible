2.3 Fit-Tier Scoring Adjustments 
The fit tier (Classic/Core/Leaning/Exploring) modifies the recommendation scoring pipeline. For lower fit tiers, the primary persona scoring bonus is reduced, which naturally promotes all_personas recommendations higher in the ranking. This ensures users with weaker persona fits receive more universally applicable content.
Fit TierPrimary Persona BonusPersona-Specific %All-Personas %Classic (75-100)+25 (unchanged)80-85%15-20%Core (50-74)+2065-70%30-35%Leaning (25-49)+1550-55%45-50%Exploring (0-24)+1030-35%65-70%Implementation: After computing the fit score, determine the fit tier, then use the corresponding primary persona bonus in Step 2 of the pipeline (Score). All other scoring components (secondary +15, barriers, goals, context, OCEAN, strength) remain unchanged.
2.4 Blend-Strength Content Rules 
Blend strength (Pure/Tendencies/Strong Influence) determines how much secondary persona content appears in each report section. These rules apply at Step 5 (Resolve Persona Context) and Step 7 (AI Synthesize).

1. Document Purpose and Scope
This document is the operational companion to the Wellness Report Template v4.0 (DOCX). It specifies, for each of the 9 report pages, exactly how content is sourced, derived, and generated. Every section includes three components:
* Data Source: Which columns, rows, or computed values feed the section.
* Derivation Logic: Formulas, filters, scoring rules, and selection criteria.
* AI Synthesis Prompt: The exact prompt to pass to the AI model to generate user-facing text.
The AI synthesis engine's role is to compose and structure pre-personalized content, not to personalize generic recommendations. The recommendation master v1.9 and persona context columns already contain persona-specific text. The AI takes this pre-personalized content and weaves it into a coherent, emotionally intelligent narrative. The AI must not add, invent, or extrapolate beyond the provided data.
NOTE: All prompts in this document assume a system-level instruction that sets the AI's role as a wellness report writer. The prompts below are the user-turn messages sent per section. See Section 13 for the system prompt.


2. System Architecture Overview
2.1 Report Generation Pipeline
The report generation follows this sequence: Assessment Complete > OCEAN Scores Calculated > Persona Assigned > Recommendations Filtered & Scored > Persona Context Resolved > Pages Generated (AI Synthesis) > Report Assembled (DOCX).
2.2 Engine Pipeline (from Implementation Guide v2.0)
StepDescription1. FilterRemove recs where user's persona is in the exclude column2. ScoreApply scoring formula: Primary +25, Secondary +15, all_personas +10, Barrier +12 (cap 36), Goal +8 (cap 24), Context +3 (cap 15), OCEAN +4 (cap 20), Strength bonus. Max = 155.3. RankSort by total score descending within each pillar4. ClusterGroup by category within pillar5. Resolve Persona ContextFor each rec, pull text from the user's primary persona column (e.g., turtle, fox, deer, bear, elephant, wolf). If empty, use the rec column as fallback.6. Select for ReportTop N recs per pillar based on score and type distribution7. AI SynthesizePass selected recs + persona context to AI for each report page

The persona title combines fit tier and blend strength:
Step 1: Calculate Fit Score (0-100)
fit_score = (1 - (user_distance / max_inter_centroid_distance)) * 100, clamped to 0-100. max_inter_centroid_distance is the largest Euclidean distance between any two persona centroids (pre-computed constant).
Step 2: Determine Fit Tier
Score RangeTierExample75-100Classic"Classic Steady Elephant"50-74Core"Core Steady Elephant"25-49Leaning"Leaning Steady Elephant"0-24Exploring"Exploring Steady Elephant"Step 3: Determine Blend Strength
blend_ratio = secondary_distance / primary_distance

PI-type recommendations (success_condition, strength_insight) are not scored. 7.3 Trait Deviation Logic
For each of the 5 OCEAN traits, compute |user_score - centroid_score|. If the deviation exceeds 0.8, generate a trait deviation card. Maximum 2 cards shown. If more than 2 traits exceed 0.8, keep only the 2 with the largest absolute deviation. Use user-facing trait names: Openness, Discipline (not Conscientiousness), Social Energy (not Extraversion), Agreeableness, Stress Sensitivity (not Neuroticism).
7.4 AI Synthesis Prompt
AI SYNTHESIS PROMPT
PROMPT: PRIMARY_PATTERN


INPUT:
- primary_persona: {persona_name}
- success_condition_text: {persona_rec text from success_condition rec for this persona}
- strength_insight_text: {persona_rec text from strength_insight rec for this persona}
- ocean_scores: {O: x.x, C: x.x, E: x.x, A: x.x, N: x.x}
- centroid_scores: {O: x.x, C: x.x, E: x.x, A: x.x, N: x.x}
- goals: [{goal_1}, {goal_2}, ...]
- barriers: [{barrier_1}, {barrier_2}, ...]
- fit_tier: {tier} (Classic/Core/Leaning/Exploring)


TASK:
Generate the following sections for the user's primary wellness personality.


SECTION 1: PERSONA NARRATIVE (150-200 words)
Write a description of how this personality pattern shows up in the user's wellness behaviour. Use the success_condition_text and strength_insight_text as the factual basis. Connect to the user's goals and barriers where the data supports it. Do not add wellness advice not present in the input.


SECTION 2: SIX BEHAVIOURAL BOXES
For each of the 6 categories below, write 2-3 sentences. Base each box strictly on the success_condition_text, strength_insight_text, and OCEAN scores provided. Do not invent behavioural claims not supported by the input data.


Box 1: Behavioural Tendencies - What patterns does this personality typically follow in wellness? Source: success_condition_text + high/low OCEAN trait signals.
Box 2: What Motivates You - What drives this personality to engage with wellness? Source: strength_insight_text + goals.
Box 3: What Drains You - What causes this personality to disengage from wellness? Source: barriers + low/unfavourable OCEAN trait signals.
Box 4: Default Under Stress - How does this personality behave when overwhelmed? Source: Stress Sensitivity score + barriers + success_condition_text.
Box 5: How You Build Habits - What habit-building approach works for this personality? Source: Discipline score + success_condition_text.
Box 6: Growth Pattern - How does this personality improve over time? Source: strength_insight_text + Openness score.


SECTION 3: TRAIT DEVIATIONS (0, 1, or 2 cards)
For each OCEAN trait where |user_score - centroid_score| > 0.8, write a 2-sentence card explaining what this deviation means for this persona. Maximum 2 cards (keep those with largest deviation). If no trait exceeds 0.8, output nothing for this section. Use user-facing trait names: Openness, Discipline, Social Energy, Agreeableness, Stress Sensitivity.


RULES:
1. Do not mention OCEAN traits by technical name. Use user-facing names only.
2. Do not mention persona names or animal names.
3. Write in second person. Use simple English.
4. Every claim must trace to the input data. Do not add wellness advice.
5. Tone: warm, insightful. Emotional arc: Deep Understanding.


OUTPUT FORMAT (strict JSON):
{ "persona_narrative": "string (150-200 words)",
  "behavioural_boxes": [ { "title": "string", "content": "string (2-3 sentences)" } x 6 ],
  "trait_deviations": [ { "trait": "string", "direction": "higher|lower", "content": "string (2 sentences)" } x 0-2 ]
}

13. AI Synthesis Engine Specifications
13.1 System Prompt
SYSTEM PROMPT (SET ONCE PER REPORT GENERATION)
You are Pausibl's wellness report writer. You compose personalized wellness reports from pre-analyzed data. Your role is NOT to personalize generic advice - the data you receive is already personalized. Your role is to compose and structure this pre-personalized content into a coherent, emotionally intelligent narrative.

CORE PRINCIPLES:
1. You are a perceptive, warm wellness coach writing a report for one specific person.
2. Every statement should feel like it was written specifically for THIS user.
3. Use simple English understandable by any adult. No academic jargon. No western wellness buzzwords.
4. Never mention OCEAN, personality traits by technical names, or persona animal names.
5. OCEAN user-facing labels: Openness, Discipline (not Conscientiousness), Social Energy (not Extraversion), Agreeableness, Stress Sensitivity (not Neuroticism).
6. Fit tier names: Classic, Core, Leaning, Exploring. Never use the old names Adaptive or Emerging.
7. Never use motivational fluff ("You've got this!", "Stay positive!", "Believe in yourself!").
8. Be behaviorally specific. "Walk for 20 minutes after dinner" not "Be more active."
9. Frame weaknesses as patterns to notice, not character flaws.
10. Frame strengths as genuine advantages, not consolation prizes.

TONE: Warm, insightful, slightly surprising. The user should feel understood, not lectured.

EMOTIONAL ARC ACROSS SECTIONS:
Page 3: Pride and Recognition ("I see my pattern clearly")
Page 4: Deep Understanding ("This report truly understands how I operate")
Page 5: Nuance ("There is more to me than one label")
Page 6: Revelation ("I never thought about it that way")
Page 7: Confidence ("I know exactly what to do")
Page 8: Excitement ("I know exactly where to start")

FIT TIER ADJUSTMENTS:
You will receive a fit_tier parameter with each section prompt. Adjust your language:
- Classic: Write with full confidence. "You naturally..." / "You always..."
- Core: Write with soft confidence. "You tend to..." / "Your pattern suggests..."
- Leaning: Write with exploration. "You may find..." / "People like you often..."
- Exploring: Write with invitation. "Some of your responses suggest..." / "Consider whether..."

BLEND STRENGTH ADJUSTMENTS:
You will receive a blend_strength parameter. Adjust content:
- Pure: Never mention a secondary persona. Write as if the user is purely one type.
- Tendencies: Include one sentence per section acknowledging the secondary influence.
- Strong Influence: Dedicate substantive content to the blend. Show how two sides interact.
