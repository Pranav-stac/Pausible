2.3 Fit-Tier Scoring Adjustments (New in v1.1)
The fit tier (Classic/Core/Adaptive/Emerging) modifies the recommendation scoring pipeline. For lower fit tiers, the primary persona scoring bonus is reduced, which naturally promotes all_personas recommendations higher in the ranking. This ensures users with weaker persona fits receive more universally applicable content.
Fit TierPrimary Persona BonusPersona-Specific %All-Personas %Classic (75-100)+25 (unchanged)80-85%15-20%Core (50-74)+2065-70%30-35%Adaptive (25-49)+1550-55%45-50%Emerging (0-24)+1030-35%65-70%Implementation: After computing the fit score, determine the fit tier, then use the corresponding primary persona bonus in Step 2 of the pipeline (Score). All other scoring components (secondary +15, barriers, goals, context, OCEAN, strength) remain unchanged.
2.4 Blend-Strength Content Rules (New in v1.1)
Blend strength (Pure/Tendencies/Strong Influence) determines how much secondary persona content appears in each report section. These rules apply at Step 5 (Resolve Persona Context) and Step 7 (AI Synthesize).
Report SectionPure (ratio > 2.0)Tendencies (1.3-2.0)Strong Influence (< 1.3)Slide 2: PersonalityPrimary only1 sentence on secondaryFull paragraph on blendSlide 3: Blind SpotsPrimary blind_spot + pattern_prediction onlyPrimary + optionally secondary pattern_predictionBoth personas' blind_spot insights, framed as two sidesSlide 4: Success BlueprintPrimary onlyPrimary + brief secondary mentionBoth personas' success_condition, framed as dual requirementSlide 5: Where You StandNo secondary mentionSecondary shown in distribution onlySecondary highlighted in distribution + narrativeSlides 7-8: Action PlanPrimary persona context onlyPrimary context, 1-2 secondary notesPrimary context + secondary "also consider" notesSlide 10: Coaching GuidePrimary PI onlyPrimary PI + secondary tendency noteExplicit blend coaching — how two sides conflict/complementAI Synthesis Impact: Every AI synthesis prompt now receives fit_tier and blend_strength as input parameters. The system prompt instructs the model to adjust tone (assertive for Classic, tentative for Emerging) and content inclusion (single-persona for Pure, dual-persona for Strong Influence).

1. Document Purpose and Scope
This document is the operational companion to the Wellness Report Template (PPTX). It specifies, for each of the 10 report slides, exactly how content is sourced, derived, and generated. Every section includes three components:
* Data Source: Which columns, rows, or computed values feed the section.
* Derivation Logic: Formulas, filters, scoring rules, and selection criteria.
* AI Synthesis Prompt: The exact prompt to pass to the AI model to generate user-facing text.
The AI synthesis engine's role is to compose and structure pre-personalized content, not to personalize generic recommendations. The recommendation master v1.7 and persona context columns already contain persona-specific text. The AI takes this pre-personalized content and weaves it into a coherent, emotionally intelligent narrative.
NOTE: All prompts in this document assume a system-level instruction that sets the AI's role as a wellness report writer. The prompts below are the user-turn messages sent per section. See Section 13 for the system prompt.


2. System Architecture Overview
2.1 Report Generation Pipeline
The report generation follows this sequence: Assessment Complete > OCEAN Scores Calculated > Persona Assigned > Recommendations Filtered & Scored > Persona Context Resolved > Sections Generated (AI Synthesis) > Report Assembled (PPTX).
2.2 Engine Pipeline (from Implementation Guide v2.0)
StepDescription1. FilterRemove recs where user's persona is in the exclude column2. ScoreApply scoring formula: Primary +25, Secondary +15, all_personas +10, Barrier +12 (cap 36), Goal +8 (cap 24), Context +3 (cap 15), OCEAN +4 (cap 20), Strength bonus. Max = 145.3. RankSort by total score descending within each pillar4. ClusterGroup by category within pillar5. Resolve Persona ContextFor each rec, pull text from the user's primary persona column (e.g., turtle, fox, deer, bear, elephant, wolf). If empty, use the rec column as fallback.6. Select for ReportTop N recs per pillar based on score and type distribution7. AI SynthesizePass selected recs + persona context to AI for each report section

The persona title combines fit tier and blend strength:
Step 1: Calculate Fit Score (0-100)
fit_score = (1 - (user_distance / max_inter_centroid_distance)) * 100, clamped to 0-100. max_inter_centroid_distance is the largest Euclidean distance between any two persona centroids (pre-computed constant).
Step 2: Determine Fit Tier
Score RangeTierExample75-100Classic"Classic Steady Elephant"50-74Core"Core Steady Elephant"25-49Adaptive"Adaptive Steady Elephant"0-24Emerging"Emerging Steady Elephant"Step 3: Determine Blend Strength
blend_ratio = secondary_distance / primary_distance
RatioLabelSuffix> 2.0PureNo suffix1.3-2.0Tendencies"...with Curious Fox tendencies"< 1.3Strong influence"...with strong Curious Fox influence"Step 4: Assemble Title
Format: "[Tier] [Primary Persona] [Blend Suffix]". Example: "Classic Steady Elephant with Steadfast Bear tendencies"

AI SYNTHESIS PROMPT
You are writing the Wellness Personality section of a personalized wellness report.

INPUT DATA:
- Primary persona: {primary_persona} ({fit_score}% fit)
- Secondary persona: {secondary_persona}
- Persona title: {persona_title}
- Archetype: {archetype_description}
- User OCEAN scores: O={o}, C={c}, E={e}, A={a}, N={n}
- Centroid values: O={co}, C={cc}, E={ce}, A={ca}, N={cn}
- User goals: {goals_list}
- User barriers: {barriers_list}

TASK:
Write a 2-3 paragraph personality narrative (150-200 words) that describes how this person approaches wellness. Do NOT list traits. Write as if you are a perceptive coach who has been observing this person.

RULES:
1. Open with a strength-based observation (emotional arc: Pride)
2. Describe their natural behavioral patterns around health, fitness, and nutrition
3. Mention their likely relationship with consistency, motivation, and environment
4. Reference their specific barriers naturally, not as a clinical list
5. End with a recognition statement that makes the user feel understood
6. Use simple English. No jargon. No academic terms.
7. Write in second person ("You tend to..." not "This persona...")
8. Do NOT mention OCEAN traits by name (no "Your Openness score...")
9. Do NOT mention persona names (no "As a Steady Elephant...")
10. Tone: warm, insightful, slightly surprising

OUTPUT FORMAT:
Return only the narrative paragraphs. No headings, no bullet points.
- Fit tier: {fit_tier} (Classic/Core/Adaptive/Emerging)
- Blend strength: {blend_strength} (Pure/Tendencies/Strong Influence)
- Secondary persona (if blend != Pure): {secondary_persona}

For each of the 5 OCEAN traits, calculate: deviation = |user_score - centroid_score|. Select traits where deviation > 0.8. Maximum 2 deviations shown. If more than 2 qualify, take the 2 largest deviations.

You are Pausibl's wellness report writer. You compose personalized wellness reports from pre-analyzed data. Your role is NOT to personalize generic advice - the data you receive is already personalized. Your role is to compose and structure this pre-personalized content into a coherent, emotionally intelligent narrative.

CORE PRINCIPLES:
1. You are a perceptive, warm wellness coach writing a report for one specific person.
2. Every statement should feel like it was written specifically for THIS user.
3. Use simple English understandable by any adult. No academic jargon. No western wellness buzzwords.
4. Never mention OCEAN, personality traits by technical names, or persona animal names.
5. Never use motivational fluff ("You've got this!", "Stay positive!", "Believe in yourself!").
6. Be behaviorally specific. "Walk for 20 minutes after dinner" not "Be more active."
7. Frame weaknesses as patterns to notice, not character flaws.
8. Frame strengths as genuine advantages, not consolation prizes.

TONE: Warm, insightful, slightly surprising. The user should feel understood, not lectured.
